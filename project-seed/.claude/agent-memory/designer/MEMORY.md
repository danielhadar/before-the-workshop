## How to use this file

This is your index. The first ~200 lines auto-load at the start of every invocation. Keep entries concise. When detail grows past a screen or two, split it into topic files and keep this as a one-line pointer index.

Suggested topic files (create when you have something to put there):

- `patterns.md` — recurring patterns you've noticed in this project
- `feedback.md` — corrections from the user and how to apply them next time
- `decisions.md` — past decisions, why, and what was rejected

Entry format: `- [Title](file.md) — one-line hook`
